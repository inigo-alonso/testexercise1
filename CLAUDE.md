# RAPB Workspace

This is a workspace for the Rapid Agentic Process Builder (RAPB).

## Purpose

This workspace is used to define and solve problems for YAML-based Ark agent development.

## CXA Workflow

Use the CXA commands to develop Ark agents:

```
/cxa:define [description]     # Define a new problem
/cxa:refine [issue-name]      # Refine existing problem definition
/cxa:implement [issue-name]   # Implement the solution
/cxa:iterate [issue-name]     # Iterate on existing solution
/cxa:validate [folder]        # Validate YAML files
```

## Structure

- `issues/NN-issue-name/` - Problem definitions and solutions (generated by CXA)
  - `problem.md` - Problem definition
  - `implementation-plan.md` - Selected approach with YAML snippets
  - `solution-documentation.md` - Final documentation with Mermaid diagram
- `implementation/` - Generated Ark YAML files (output from CXA, shared across all issues)
  - `agents/` - Generated agent YAML files
  - `tools/` - Generated tool YAML files
  - `queries/` - Generated query YAML files
  - `teams/` - Generated team YAML files
  - `models/` - Model configuration files
  - `secrets/` - Secret files
  - `mcpservers/` - MCP server configurations
- `files/` - Files storage for input and output files
  - `input/` - Uploaded files (PDFs, documents, etc.) - do not edit
  - `output/` - Output files generated by agents - do NOT edit

## Excel API MCPServer Naming - CRITICAL

**When creating MCPServers that use the Excel API (reference `tool-excel-api` service), you MUST use system-specific names.**

### Reserved Names - DO NOT USE
These names are reserved for central infrastructure and will cause deployment failures:
- `excel-api` - RESERVED
- `filesystem` - RESERVED
- `docs` - RESERVED

### Correct Naming Pattern

Each Excel "system" needs its own MCPServer with a **descriptive, system-specific name**:

| Business Context | MCPServer Name | Auto-Created Tool |
|-----------------|----------------|-------------------|
| Sales leads data | `leads` | `leads-excel-set-filepath` |
| Order management | `order-manager` | `order-manager-excel-set-filepath` |
| Customer tracking | `customer-tracker` | `customer-tracker-excel-set-filepath` |
| ERP system data | `erp-data` | `erp-data-excel-set-filepath` |

### Example - CORRECT MCPServer YAML

```yaml
apiVersion: ark.mckinsey.com/v1alpha1
kind: MCPServer
metadata:
  name: sales-leads  # System-specific name, NOT "excel-api"
  namespace: exercise-rfx-procurement-mmxoipnj-p6va6n
spec:
  address:
    valueFrom:
      serviceRef:
        name: tool-excel-api  # References central service (correct)
        namespace: rapp
        port: "http"
  path: "/mcp"
  timeout: "60s"
  transport: http
  description: "Sales Leads System (Excel-backed)"
  session:
    - tool: sales-leads-excel-set-filepath  # MUST be: {mcpserver-name}-excel-set-filepath
      parameters:
        - name: filepath
          value: "files/input/sales-leads.xlsx"
```

**Reference**: See `agent-lib/excel-api/CLAUDE.md` for complete Excel API pattern documentation.

### MCP Tool YAML Template (for filesystem/MCP operations)
```yaml
apiVersion: ark.mckinsey.com/v1alpha1
kind: Tool
metadata:
  name: filesystem-write-file
  namespace: exercise-rfx-procurement-mmxoipnj-p6va6n
spec:
  type: mcp
  description: "Write content to a file"
  inputSchema:
    type: object
    properties:
      path:
        type: string
        description: "Path to the file (relative to workspace)"
      content:
        type: string
        description: "Content to write"
    required: ["path", "content"]
  mcp:
    serverRef:
      name: filesystem
      namespace: exercise-rfx-procurement-mmxoipnj-p6va6n  # MUST be workspace namespace, NOT default
    toolName: write_file  # MUST use snake_case (write_file, read_file, list_directory)
```

**⚠️ MCP Tool Critical Rules:**
- `toolName` MUST use `snake_case`: `write_file`, `read_file`, `list_directory`, `set_base_directory`
- `serverRef.namespace` MUST be the workspace namespace (`exercise-rfx-procurement-mmxoipnj-p6va6n`), NOT `default`

### Filesystem Paths in Agent Prompts

When your agent needs to read/write files, use the **correct relative paths**:

```yaml
spec:
  prompt: |
    ...
    1. Read input from files/input/data.json     # ✅ Input files
    2. Process the data...
    3. Save the report to files/output/report.json  # ✅ Output files go in files/output/
    ...
```

**Path Conventions:**
| Purpose | Path | Example |
|---------|------|---------|
| Input files (user uploads) | `files/input/` | `files/input/customer-data.json` |
| Output files (agent-generated) | `files/output/` | `files/output/analysis-report.json` |

**⚠️ DO NOT use `input/` or `output/` directly** - always use `files/input/` and `files/output/` for proper file organization.

**DO NOT use absolute paths** like `/data/...` or `exercise-rfx-procurement-mmxoipnj-p6va6n/output/` in the prompt. The MCPServer.spec.session automatically calls `set_base_directory` with the correct workspace, setting the filesystem base directory before the agent runs.

### Common Mistakes to AVOID
| ❌ WRONG | ✅ CORRECT |
|----------|-----------|
| `apiVersion: ark.ai/v1` | `apiVersion: ark.mckinsey.com/v1alpha1` |
| `instruction: ...` | `prompt: ...` |
| `model: gpt-4` | `modelRef: { name: default }` |
| `tools: [tool-name]` | `tools: [{ name: tool-name }]` |
| `displayName: ...` | `description: ...` |
| `type: function` | `type: fetcher` or `type: mcp` |
| `type: http` | `type: fetcher` |
| `toolName: write-file` (kebab-case) | `toolName: write_file` (snake_case!) |
| `serverRef.namespace: default` | `serverRef.namespace: exercise-rfx-procurement-mmxoipnj-p6va6n` |
| Prompt: `save to exercise-rfx-procurement-mmxoipnj-p6va6n/output/` | Prompt: `save to files/output/` (relative!) |
| Prompt: `read from input/data.json` | Prompt: `read from files/input/data.json` |
| Prompt: `save to output/report.json` | Prompt: `save to files/output/report.json` |
| `spec.prompt` in Query | `spec.input` in Query |
| `namespace` in `spec.targets[]` | Remove - targets inherit from Query metadata |
