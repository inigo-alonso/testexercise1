# RFx readiness HTML refactor — change log

Substantive refactor of `run_analysis.py` (analysis + HTML generator) to align with the exercise master prompt: **three decision modules**, **audit-grade traceability**, **supplier hierarchy**, and **derived confidence** (not decorative UI).

## Summary of changes

| Area | Change |
|------|--------|
| **Analysis** | Replaced flat readiness+RFx+supplier logic with `analyze_full()`: Module 1 (PR quote-enablement only), Module 2 (contract-based RFx routing), Module 3 (exact → similar → capability buckets + transparent score). |
| **Readiness** | Removed contract from readiness criteria. Readiness = drawing, Authority Grp AUTO, mandatory PR fields only. Status: `READY` / `AT RISK` / `BLOCKED`. |
| **RFx routing** | Dedicated routing table on `SAP_PR.Contract`: populated → `RFx NOT REQUIRED`; empty → `RFx REQUIRED` with explicit rationale (no redundant “confirm absence” when blank is unambiguous). |
| **Suppliers** | Hierarchical buckets A/B/C; scored ranking; **Primary** (top tier), **Secondary**, **Watchlist** (excerpt); “not promoted higher” column; capability expansion flagged via `capability_affected`. |
| **Similarity** | On-page ordered list: same part → same XPAC triple on other SAP_PR parts → Preferred_vendors capability match. |
| **Executive summary** | Go/Stop/Fix, RFx headline chips, severity, overall confidence + one-line derivation, primary invitee count. |
| **Risks & follow-ups** | Structured tables with severity (Critical/Moderate/Informational) and ownership columns (`Blocks RFx?`, `Related decision module`). |
| **Data & confidence** | Per-module sources + **derived** confidence rationale (not generic “High because Excel”). |
| **Audit & validation** | Decision path string, rules list, validation checklist including **supplier count consistency** (primary + secondary + watchlist total vs excerpt), **excluded unsupported claims** line. |
| **Commercial context** | Small labeled section for exercise quote files only (`CONTEXT` badge); not used for supplier qualification. |
| **UI cleanup** | Removed mass mailto / pseudo workflow actions; Chart.js only when PO stats exist; navy bar colors aligned. |

## Mapping: old issue → fix → exercise alignment

| Old issue | Fix implemented | Why it aligns |
|-----------|-----------------|---------------|
| Readiness mixed with contract (“2/3” confusion) | Contract only in Module 2; readiness table excludes contract | Exercise requires **clean separation** of quote readiness vs RFx routing. |
| All suppliers presented as equal | Scoring (tier base + recency + order count), roles Primary/Secondary/Watchlist | Demonstrates **ranked decision logic**, not a flat list. |
| Weak similar-part / capability story | Bucket B from SAP_PR XPAC match + PO; Bucket C from Preferred_vendors minus A/B | **Hierarchy exact → similar → capability** is explicit in the artifact. |
| “High confidence” without derivation | `conf_readiness_why`, `conf_rfx_why`, `conf_supplier_why`, overall composite | Matches **hallucination/validation** guidance: confidence must be justified. |
| No audit trail | Decision path, rules applied, validation checks, FACT badges on tables | **Evaluator can trace** PR → rules → outcome. |
| Spurious follow-ups | No “confirm contract” when field is confidently empty; structured owners | **Ownership-ready** triage per partner-review standard. |
| Supplier count drift | Validation row + caption distinguishes watchlist **total** vs table **excerpt** | **Consistency** across summary, tables, validation. |
| Preferred vendors claimed but not shown | `capability_affected` + claims text when PV rows exist but don’t change primary set | Honest about **whether capability tier mattered**. |
| Decorative email buttons | Removed from HTML output | Focus on **reasoning quality** over workflow chrome. |
| Optional supplier offers | Supplemental commercial table, CONTEXT-only | Offers as **optional commercial context**, not historical qualification. |
